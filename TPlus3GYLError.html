T3供应链结转失败原因分析


调查：

    跟踪发现在为该表dispatchlist插入某条记录时，结转结束；分析这条插入记录发现表dispatchlist的cMome字段长度为60，而插入的记录长度达到了87个字符，数据库报错”将截断字符串或二进制数据！“。


解决方案：

    执行以下SQL语句

    alter table UFDATA_007_2015.dbo.DispatchList alter column [cMemo] [varchar] (600) COLLATE Chinese_PRC_CI_AS NULL;

    alter table UFDATA_007_2015.dbo.DispatchLists alter column [cMemo] [varchar] (600) COLLATE Chinese_PRC_CI_AS NULL;

    alter table UFDATA_007_2015.dbo.ScrapVouch alter column [cMemo] [varchar] (300) COLLATE Chinese_PRC_CI_AS NULL;

    alter table UFDATA_007_2015.dbo.SO_SOMain alter column [cMemo] [varchar] (600) COLLATE Chinese_PRC_CI_AS NULL;

    将表DispatchList 、DispatchLists 、ScrapVouch 、SO_SOMain 的备注字段均不同程度的放大即可。
